# 灵兽品质系统说明

## 品质等级体系

游戏中所有灵兽按照三个品质等级划分：

### 🟦 凡品 (Common Tier)
- **稀有度**: 1-5星
- **适用境界**: 炼气期至筑基期
- **特点**: 基础属性，易于获得，适合新手
- **数量**: 10种

### 🟪 仙品 (Immortal Tier)
- **稀有度**: 6-8星
- **适用境界**: 结丹期至元婴期
- **特点**: 强力属性，有特殊效果，较难获得
- **数量**: 10种

### 🟨 神品 (Divine Tier)
- **稀有度**: 9-10星
- **适用境界**: 化神期及以上
- **特点**: 极致属性，传说特效，极其稀有
- **数量**: 10种

## 设计理念

### 品质递进
- **凡品**: 单一效果，简单直接，适合新手上手
- **仙品**: 复合效果，具有战术价值，需要策略运用
- **神品**: 颠覆战局，具有传说特效，改变战斗方式

### 获取难度
- **凡品**: 野外捕捉、日常获取，容易培养
- **仙品**: 秘境探索、积分兑换，需要投入
- **神品**: 极限挑战、限量商品，极其珍贵

### 培养价值
所有灵兽都有培养价值，但神品灵兽的成长上限最高，适合作为终极战力。

## 数据库字段说明

在 `spirit_beast_templates` 表中新增 `quality` 字段：

```sql
quality VARCHAR(20) DEFAULT '凡品' NOT NULL
-- 取值: '凡品', '仙品', '神品'
```

### 品质与稀有度对应关系
```sql
-- 凡品：1-5星
UPDATE spirit_beast_templates SET quality = '凡品' WHERE rarity BETWEEN 1 AND 5;

-- 仙品：6-8星
UPDATE spirit_beast_templates SET quality = '仙品' WHERE rarity BETWEEN 6 AND 8;

-- 神品：9-10星
UPDATE spirit_beast_templates SET quality = '神品' WHERE rarity BETWEEN 9 AND 10;
```

## 迁移说明

### 新部署
执行 `data/init_spirit_beasts.sql` 初始化所有30种灵兽数据

### 现有数据库
执行 `data/migrations/add_beast_quality.sql` 添加品质字段并自动设置

```bash
# 备份数据库
cp data/xiuxian.db data/xiuxian.db.backup.$(date +%Y%m%d)

# 执行迁移
sqlite3 data/xiuxian.db < data/migrations/add_beast_quality.sql
```
